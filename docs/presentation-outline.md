# Présentation & Vidéo – Notes App

## Pitch deck (≤ 10 slides)

1. **Titre & Contexte** – Objectif du projet, stack principale.
2. **Problème utilisateur** – Pain points autour de la prise de notes multi-contextes.
3. **Solution** – Aperçu global de l’application (backend + frontend + DevOps).
4. **Fonctionnalités clés** – Auth JWT, CRUD, filtres, pièces jointes, stats.
5. **Architecture** – Schéma (Mongo ↔ Express ↔ React) + pipelines CI/CD.
6. **UX/UI** – Captures light/dark mode, TinyMCE, dashboard.
7. **Sécurité & Qualité** – Validation, sanitization, tests Jest (80% coverage).
8. **DevOps** – Docker Compose, GitHub Actions, variables d’environnement.
9. **Plan de projet** – Board Trello, user stories, workflow PR.
10. **Roadmap & Next steps** – Idées futures (partage, notifications, mobile).

## Vidéo démo (3–5 minutes)

Script suggéré :

1. **Intro (20s)** – Présentation rapide, objectif de l’app.
2. **Auth (30s)** – Inscription + connexion, persistance du token.
3. **Création note (60s)** – TinyMCE, upload fichier, catégories colorées.
4. **Dashboard (60s)** – Recherche en direct, filtres, stats, pagination, mode sombre.
5. **Edition + suppression (40s)** – Mise à jour d’une note, nouvelle pièce jointe, modal de suppression.
6. **Backend / Swagger (30s)** – Aperçu rapide de l’API et des tests (npm test).
7. **DevOps (20s)** – docker compose up, mention CI GitHub Actions.
8. **Outro (20s)** – Rappel des livrables, roadmap.
